Sales Insights - Data Cleaning, Data Transformation, Data Visualization

Objective:

To contribute to the success of a business by utilizing data analysis techniques, specifically focusing on time series analysis, to provide valuable insights and accurate sales forecasting. The data is of US only.

Description

The objective can be broken down into the following detailed components:

  1. Dashboard Creation:Identify the KPIs, design an intuitive and visually appealing dashboard, add interactive visualizations and filtering capabilities to allow users to explore the data at various levels of granularity.
  2. Data Analysis:Provide valuable insights to business entities regarding the effectiveness of their sales strategies through visualization and charts.
  3. Sales Forecasting: Leverage historic data and apply time series to generate sales forecasts for next 15 days.
  4. Actionable Insights and Recommendation: End goal is to share valuable insights and actionable information that can drive strategic decision-making and support the supermarket’s goals for growth, efficiency, and customer satisfaction.

DAX Queries:

Go to data view and add a new column from there where we will going to perform DAX queries in order to solve our problem statement.

In this case I am taking “AvgDelivery” that will provide average delivery time to reach the product to the customer. Now in case if a company wanted to know the average delivery rate to analyze their service, they can see this directly from the visualization.

AvgDelivery = DATEDIF(‘SuperStore_Sales’[Order Date’], ‘SuperStore’ [Ship Date], DAY) This will give us the number of days the order takes to deliver. Now to show the average rate of delivery days, we can use KPI to get the average delivery days.

Forecasting:

Forecasting is the process where we get the future expectations or future results based on current Create a new sheet for forecasting, for forcasting we have to first create a line chart, forcasting is based on time series, so we will use order date and sales, then we will get a straight line having data based on year, we can customize is by removing year then we will get month based chart, after removing month we will get more specific days based chart. In this case we will get a 30 days data. We can change it into order date from dropdown and then it will show all the data of 2 years per day. For forcasting, go to the the add further analysis third icon and then select forcast option from there. Customize the days, ignore the last that will ignore the data of last date, confidence interval, after this we will get forcast upper and lower value, this will provide forcast value of 15 days, and we can export it or show as table too.

Table Dax:

In forecasting process, we will have duplicate order dates and in such case Power BI is using aggregate method itself. Now if we want to have single and unique result using single values only, this will be done using DAX query. We will go to data view, then table tools, click on new table, then name the table SalesForecast, we will use orderdate and sales column, use formula summarize, and write the table name, then column name and one more attribute of sales there from the same table name. Now we get the table from there. Now we will get unique dates only and it will aggregate all the sales accordingly.
